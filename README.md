# AutoText2SQL Agent PoC

PoC агентной системы для поиска и навигации по фиксированному набору реляционных баз данных: схемам, таблицам, колонкам, связям и базовым примерам данных.

## Что за задача, для кого и какая боль сейчас

### Проблема
- В нескольких корпоративных БД сложно быстро найти, где лежат нужные данные: какая таблица отвечает за сущность, какие поля использовать и как таблицы связаны.
- Аналитики, тестировщики и системные аналитики тратят много времени на ручной разбор схем и консультации с дата-инженерами.
- Нужен единый интерфейс поиска по структуре конкретного списка БД, с учетом ограничений доступа.

### Для кого
- Аналитики и product owner.
- Тестировщики, которые ищут данные для проверки сценариев.
- Системные аналитики и UX-дизайнеры при подготовке требований.

## Что именно сделает PoC на демо

PoC покажет сквозной агентный контур для поиска по структуре конкретных БД:
- Подключение к заранее заданному набору БД и сбор метаданных (схемы, таблицы, колонки, ключи, связи).
- Индексация метаданных в едином поисковом слое (knowledge base).
- Поиск по естественному языку: "где хранится X", "в каких таблицах есть Y", "как связаны A и B".
- Выдача структурированного ответа: БД -> схема -> таблица -> колонки -> связи + краткое объяснение.

## Что НЕ делает PoC (out-of-scope)

- Поддержку всех СУБД и SQL-диалектов без ограничений.
- Полный legal/compliance контур для всех отраслевых стандартов.
- Автоматическое выполнение DDL/DML-операций и любых изменений данных.

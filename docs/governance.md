# Governance: AutoText2SQL Agent PoC

## 1) Risk Register

| Риск | Вероятность / Влияние | Детект | Защита | Остаточный риск |
|---|---|---|---|---|
| Нерелевантный поиск (не те таблицы/колонки) | Средняя / Высокое | Низкий Precision@k, пользовательские жалобы | Retrieval+rеranking, словарь терминов, eval-набор с edge-кейсами | Средний |
| Prompt/metadata injection через пользовательский ввод | Высокая / Высокое | Policy violations, аномальные шаблоны запроса | Input sanitization, изоляция системных инструкций, denylist паттернов | Средний |
| SQL injection и попытка модификации данных через сгенерированный SQL | Средняя / Критическое | Обнаружение небезопасных SQL-паттернов, срабатывания policy-валидатора, аномалии в аудит-логах БД, подтверждение действий пользователем перед их выполнением | Соединение с БД открывается в режиме `read only` | Низкий |
| Устаревший индекс метаданных | Средняя / Среднее | Падение релевантности, drift detection | Регулярная переиндексация, schema-change alerts, TTL индекса | Средний |
| Неоднозначность бизнес-терминов | Высокая / Среднее | Частые уточнения или неправильные выборы таблиц | Семантический словарь, уточняющие вопросы | Средний |

## 2) Политика логов и персональных данных

### Логи
- Логируются технические события поиска: этапы retrieval/ranking, коды ошибок, метрики качества и времени.
- Логируются пары запрос на человеческом языке и сформированный SQL запрос, а также список бд обьектов, в которых выполнялись запросы
- Результаты данных из БД целиком не сохраняются

### Работа с персональными данными
- Проект будет работать только с тестовыми стендами и риск утечки персональных данных отсуствует

## 3) Защита от injection и опасных действий

### Human in the loop
- Любое выполнение SQL в БД допускается только после явного подтверждения человеком.
- Агент не имеет права автоматически запускать SQL, даже если запрос формально безопасен и проходит policy-валидацию.

### Injection и небезопасный ввод
- Система работает только с DQL подмножеством языка SQL, соединение с сервером бд открывается в режиме Read Only
- Система работает только с тестовыми стендами, поэтому риск утечки персональных данных не возникает
- Контекст промптов разделяется: системные инструкции отделены от пользовательского контента.

## 4) Операционные ограничения

- Вводится бюджет эксплуатации агента (дневной и недельный лимиты затрат), по достижении лимита выполнение обработки запросов блокируется до сброса лимита.
- Для каждой задачи устанавливается верхний порог `cost-per-task`; при превышении порога агент обязан остановить выполнение и запросить подтверждение человека.
- В логах фиксируются фактическая стоимость задачи и причина остановки/эскалации при срабатывании лимитов.
